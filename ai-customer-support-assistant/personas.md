Personas :These personas were created to guide product decisions and prioritize features based on real user needs.

- Customer (Alice)
  - Description: End-user seeking help (technical or billing).
  - Goals: Fast resolution, clear instructions, minimal back-and-forth.
  - Pain points: Long hold times, confusing guidance, repeated explanations.
  - Success metrics: Time-to-resolution, CSAT, first-contact resolution rate.

- Support Agent (Bob)
  - Description: Human agent handling escalations and complex cases.
  - Goals: Faster resolution, reduced repetitive work, better context.
  - Pain points: Repetitive queries, insufficient context, tool fragmentation.
  - Success metrics: Tickets handled per shift, agent satisfaction, average handle time.

- Support Manager (Carol)
  - Description: Oversees operations and quality.
  - Goals: Maintain SLAs, optimize headcount, improve CSAT.
  - Pain points: Hard to measure coaching impact, poor analytics.
  - Success metrics: SLA compliance, cost per ticket, CSAT trends.

- Developer / ML Engineer (Devi)
  - Description: Builds and maintains the assistant.
  - Goals: Reliable pipelines, measurable model behaviour, easy integration.
  - Pain points: Data quality, annotation overhead, model drift.
  - Success metrics: Deployment frequency, error rates, model latency.

- Compliance Officer (Evan)
  - Description: Ensures legal & privacy requirements are met.
  - Goals: Auditable logs, safe responses, PII protection.
  - Pain points: Hard to trace model decisions, maintaining audit trails.
  - Success metrics: Audit readiness, PII leakage incidents, compliance sign-off.
